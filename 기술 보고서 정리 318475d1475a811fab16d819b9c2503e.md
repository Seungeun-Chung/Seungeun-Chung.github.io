# 01. Jetson Orin NX 기반 차량 번호판 실시간 인식 시스템 개발

## 1. 프로젝트 개요

- **프로젝트명:** Jetson 기반 4K 차량 번호판 실시간 인식 시스템 개발
- **기간:** 2024.09 ~ 2025.02 (기업 협업)
- **협업사:** HuNature (임베디드 AI 솔루션 기업)
- **역할 및 기여도:**
    - 차량 번호판 식별 알고리즘 설계·구현 (YOLOv8 기반)
    - DeepStream SDK 파이프라인 구축 및 Jetson Orin NX 배포
    - TensorRT 양자화 최적화 및 Docker 컨테이너화
- **사용 기술 스택:**
    - **하드웨어:** NVIDIA Jetson Orin NX (4 × A78 cores, 2048 CUDA cores)
    - **소프트웨어/프레임워크:**
        - DeepStream SDK (v6.x)
        - YOLOv8 (학습: Python, PyTorch)
        - TensorRT (FP16/INT8 양자화)
        - ByteTrack (실시간 트래킹)
        - PyTesseract (OCR)
        - OpenCV, GStreamer, Docker
- **목표 및 동기:**
    - **문제 정의:** 임베디드 환경(Jetson Orin NX)에서 4K 해상도 실시간 차량 번호판 식별
    - **성과 기준:** F1-score ≥ 90% / 실시간 처리(≈60 FPS)
    - **추가 요구사항:**
        1. DeepStream 기반 End‑to‑End 파이프라인 구성
        2. 실시간 시각화(2분할 화면)
        3. OCR 및 객체 트래킹 통합

## 2. 배경 지식

- **임베디드 AI 플랫폼 (Jetson Orin NX):**
    - 저전력·고성능 AI 추론용 SoC(System‑on‑Chip)
    - CUDA, TensorRT, DeepStream 최적화 지원
- **DeepStream SDK:**
    - NVIDIA가 제공하는 스트리밍 기반 비디오 분석 프레임워크
    - GStreamer 플러그인 형태로 입력→처리→출력 파이프라인 구성
- **객체 검출(Object Detection):**
    - 이미지 내 객체 위치(bounding box)와 클래스(label)를 예측
    - 대표 모델: YOLO 계열(You Only Look Once)
- **트래킹(Tracking):***
    - 프레임 간 객체 ID를 유지하며 이동 경로 추적
    - ByteTrack: 고정밀·고속 온라인 트래커
- **OCR(Optical Character Recognition):**
    - 이미지 내 문자영역을 탐지하고 텍스트로 변환
    - PyTesseract: Tesseract 엔진 파이썬 래퍼

## 3. 요구사항 및 설계

- **기능 요구사항:**
    1. 4K 해상도 입력 실시간 차량 및 번호판 검출
    2. 트래킹 ID 부여 및 OCR 결과 동기화
    3. 두 개의 시각화 화면 제공
- **비기능 요구사항:**
    - **실시간성:** ≥ 55 FPS 유지
    - **정확도:** F1-score ≥ 90%
    - **리소스 제약:** Jetson Orin NX 메모리/연산 한계 고려
- **시스템 아키텍처:**
    1. **Input:** 카메라→GStreamer 캡처 → DeepStream `nvstreammux`
    2. **Pre‑processing:** 4분할(또는 차량 중심) 크롭 + 마진 적용
    3. **Inference:** YOLOv8 변환 모델(TensorRT FP16/INT8)
    4. **Post‑processing:** NMS → ByteTrack 트래킹 → PyTesseract OCR
    5. **Visualization:** DeepStream `nvdsink` + OpenCV Overlay (2분할)

## 5. 알고리즘/모델 설계

- **Base Model:** YOLOv8 (기업 요구사항)
- **커스텀 기법:**
    1. **4분할 탐색:**
        - 입력 영상에 n% 마진 → 4등분 → 각 섹션을 모델 입력 사이즈로 리사이즈

    - 학습 lr = 0.001, batch = 16, epoch = 50

    ## 🔷 1. Introduction
    
    > 목표: 문제 제기, 기존 방법 한계, 제안하는 GuiFiR 프레임워크 개요, 기여 정리
    > 
    - 기존 손 포즈 추정(hand pose estimation)은 기타의 줄/프렛 상에서 손가락의 정확한 위치까지 추정하지 못함
    - 손과 기타 프렛보드의 동시 추정 필요 → 새로운 문제 정의: **기타 핑거 리딩 (Guitar Finger Reading)**
    - 데이터 부족 문제 해결:
        - **반지도 학습 (semi-supervised)**: 소량 템플릿을 합성해 대량 생성
        - **약지도 학습 (weakly-supervised)**: 오디오 기반 타블라츄어 자동 생성
    - 제안 구성:
        - `GuiFiR-Synth`: 합성 이미지 + 좌표 GT
        - `GuiFiR-Vid`: 실제 연주 영상 + pseudo-GT 타블라츄어
        - `GuiFiR-Net`: 손(HPE)과 프렛보드(FPE) 추정 네트워크
    
    > 🔹 핵심 기여 요약:
    > 
    1. 새로운 문제 정의 (기타 핑거 리딩)
    2. 기존 구성요소(HPE, FPE)를 단순 통합한 네트워크
    3. 약/반지도 학습 기법 + 멀티모달(영상+오디오) 활용
    4. 공개 데이터셋 및 종합 실험

    ## 🔷 2. Related Works
    
    > 목표: 기존 연구 카테고리화 및 GuiFiR와의 차별점 명확히 설명
    > 
    
    ### (1) Hand Pose Estimation
    
    - 일반적 HPE: 2D 또는 3D 관절 좌표 회귀 방식
    - Mesh 기반 고급 방식: 3D 손 모델을 정렬
    - 문제점: 기타처럼 복잡한 물체와 상호작용하는 상황에서는 정확도 저하
    
    ### (2) 손-물체 상호작용 기반 HPE
    
    - 여러 데이터셋 (예: H2O, ARCTIC)을 통해 연구됨
    - 하지만 기타처럼 복잡한 조작은 다루지 않음
    
    ### (3) 기타 연주에 특화된 방법
    
    - 기존 대부분은 **오디오 기반 타블라츄어 추정**
    - 영상 기반 방법은 RGB-D 센서 등 특수 장비 필요하거나, 프렛보드가 손에 가려 정확하지 않음
    - **GuiFiR의 차별점**:
        - RGB 이미지만 사용
        - 손과 프렛보드의 **공동 추정** (joint estimation)

    ## 🔷 3. Datasets
    
    > 목표: GuiFiR이 사용하는 합성/실제 데이터셋 구성 및 GT 생성 방식 설명
    > 
    
    ### 3.1. GuiFiR-Synth (합성 이미지)
    
    - 4개 카테고리: 배경(2766), 사람(12), 기타(14), 손(20)
    - 기타 템플릿: 6줄 × 20프렛 + 넛(nut) → 126개 프렛보드 좌표 수동 주석
    - 손 템플릿: 왼손, 기타 연주 자세로 수집
    - 합성 방식:
        - 각 템플릿을 무작위로 조합 → 합성 이미지 생성
        - 원본 템플릿의 좌표 정보를 그대로 전이
    
    ### 3.2. GuiFiR-Vid (실제 영상)
    
    - YouTube에서 1030개의 기타 연주 클립 수집
    - 구성: 훈련(900), 검증(30), 테스트(100)
    - 수동 타블라츄어 주석: 일부 영상만 수동으로 수행
    - 수식 (Equation 1): 프렛 위치 계산식dn=s(1−e−kn)
        
        dn=s(1−e−kn)d_n = s(1 - e^{-kn})
        
        - `s`: 스케일 길이(넛~브릿지까지)
        - `n`: 프렛 번호
    - 프렛보드 코너 4점 수동 주석 → 호모그래피 변환 → 전체 프렛 좌표 계산
    
    > 🔹 포인트: 실제 영상에서는 프레임마다 손가락이 눌리고 있지 않을 수 있기 때문에, 오디오 기반 타블라츄어와 정렬된 프레임만 사용
    > 
    
    ---
    
    ## 🔷 4. Network and Training
    
    ### 4.1. Network Components
    
    ### ▪ Hand Pose Estimator (HPE)
    
    - MediaPipe, OpenPose (2D)
    - HaMeR, WiLoR (3D)
    - Fine-tuning 가능하면 어떤 HPE도 사용 가능
    
    ### ▪ Fretboard Pose Estimator (FPE)
    
    - MMPose 기반 ResNet50 사용
    - 출력: 126개 프렛보드 좌표 회귀
    - 손실함수: Smooth L1 Loss + 가중치 조절 (코너, 엣지, 내부 좌표 비중 조정)
    
    ### ▪ Detector
    
    - Grounding DINO 사용: “left hand”, “fretboard” 키워드로 감지
    - 데이터셋 외 5863개 프레임을 이용해 파인튜닝
    
    ### ▪ TabCNN
    
    - 오디오 → 타블라츄어로 변환
    - 학습에는 사용되지만 추론 단계에서는 사용 안 함
    
    ---
    
    ### 4.2. Joint Fine-Tuning
    
    > 두 데이터셋을 번갈아 학습하며, 서로 다른 손실함수 적용
    > 
    
    ### ▪ Strong supervision: `GuiFiR-Synth`
    
    Lh=∥HPE output−GT hand coords∥2L_h = \| \text{HPE output} - \text{GT hand coords} \|^2
    
    Lh=∥HPE output−GT hand coords∥2
    
    Lf=∥FPE output−GT fret coords∥2L_f = \| \text{FPE output} - \text{GT fret coords} \|^2
    
    Lf=∥FPE output−GT fret coords∥2
    
    ### ▪ Weak supervision: `GuiFiR-Vid`
    
    - 오디오로부터 추출된 frame-tab (6줄 중 누른 프렛 번호) 사용
    - 손가락 5개 좌표 ↔ 프렛보드 타겟 좌표를 **Hungarian Matching** 으로 매칭
    
    Ltab=∑matchL2 distanceL_{\text{tab}} = \sum_{\text{match}} \text{L2 distance}
    
    Ltab=match∑L2 distance
    
    ### ▪ Regularization Losses
    
    - 손가락 간 거리 제약 (좌표 일관성 보장)
    - Fretboard 내부 좌표 일관성 유지:
    
    Lregf=∥FPE output−G(4 corner points)∥2L_{\text{regf}} = \| \text{FPE output} - G(\text{4 corner points}) \|^2
    
    Lregf=∥FPE output−G(4 corner points)∥2

    ### 5.1. Implementation Details

## 2. 배경 지식

- **도메인 설명:**
    - 기타 연주자 손-프렛보드 상호작용 포즈 추정은 기존 HPE 모델이 객체 상호작용 감지를 못하는 한계가 있음
- **기본 개념:**
    - **Hand Pose Estimation (HPE):** 2D 관절 좌표 예측 (MediaPipe, OpenPose)
    - **Fretboard Pose Estimation (FPE):** 기타 프렛보드 상의 프렛·스트링 교차점 좌표 예측 (126개)
    - **Weakly & Semi-Supervised Learning:** 합성 강지도 + 실제 약지도 학습으로 데이터 효율성 강화
- 모델 설명
    - **Grounding DINO:** 객체 감지에 텍스트 프롬프트를 활용하는 최신 Detector. 학습된 Transformer 기반 백본으로, 손 또는 프렛보드와 같은 객체를 prompt 기반으로 정확히 검출할 수 있음
    - **MediaPipe:** Google에서 개발한 경량 실시간 HPE 라이브러리. 21개 손 관절의 2D 좌표를 빠르게 추정하며, 모바일/임베디드 환경에 최적화됨
    - **OpenPose:** CMU Perceptual Computing Lab에서 개발한 멀티-인물 2D 포즈 추정 모델. Part Affinity Fields(PAF)를 사용해 관절 연결을 예측하여 높은 정확도를 제공
    - **HaMeR:** Hand Mesh Recovery 모델로, 2D 영상에서 3D 손 메쉬와 관절을 함께 복원. 손-객체 상호작용 상황에서도 안정적인 포즈 예측을 위한 baseline으로 활용됨
    - **WiLoR:** Weakly-supervised Learning of Relations 모델로, 객체와 물체 간 관계 학습을 통해 포즈 예측 정확도를 높임. 제한된 라벨 데이터로도 학습 가능
    - **ResNet-50 + UNet Head (FPE):** ResNet-50 백본에서 추출한 특징맵을 UNet 구조로 복원하여 126개 프렛보드 포인트를 정확히 회귀함
    - **TabCNN:** 오디오 스펙트로그램을 입력받아 기타 탭(음표 배치)을 예측하는 CNN 기반 모델. Hungarian matching 손실을 사용해 weak supervision 타깃을 생성하고, 멀티모달 학습 루프에서 pseudo-GT로 활용됨

## 3. 요구사항 및 설계

- **기능 요구사항:**
    1. 손 관절 21점 및 프렛보드 126점 동시 추정
    2. 오디오 기반 탭 정보(TabCNN)로 약지도 타겟 생성
- **비기능 요구사항:**
    - **데이터 효율성:** 합성 이미지 56,000장, 실제 클립 1,030개 사용
    - **추론 속도:** GPU 기준 10 FPS 이상
- **시스템 아키텍처:**
    1. **Det Module:** Grounding DINO로 손/프렛보드 박스 검출
    2. **HPE Module:** MMPose 기반 손 포즈 추정 (Fine-tuned)
    3. **FPE Module:** ResNet-50 백본 + UNet 헤드로 Grid 포인트 회귀
    4. **Audio Module:** TabCNN으로 pseudo GT 탭 정보 생성
    5. **Joint Learning:** 강지도(합성) ↔ 약지도(실제) 교차 학습 루프
